# Техническое задание — LawCRM (расширенное, с мобильными приложениями)

## 1. Общая архитектура

| Компонент | Технология |
|---|---|
| Бэкенд | Python 3.10+, FastAPI |
| База данных | SQLite (dev) / PostgreSQL (prod) |
| Веб-фронтенд | React 18 + TypeScript + Vite |
| API для мобильных | REST API (JSON) на том же FastAPI |
| iOS приложение | Swift / SwiftUI |
| Android приложение | Kotlin / Jetpack Compose |
| Аутентификация | JWT токены (мобильные), сессии (веб) |
| Фоновые задачи | APScheduler |
| Пуш-уведомления | Firebase Cloud Messaging (FCM) |

---

## 2. Мобильные приложения — требования

### 2.1. Общие для iOS и Android

- Минимальная версия iOS: 15.0
- Минимальная версия Android: 8.0 (API 26)
- Поддержка тёмной темы
- Офлайн-режим: просмотр ранее загруженных дел без интернета
- Touch ID / Face ID / Biometric для быстрого входа
- Пуш-уведомления о заседаниях (за 3 дня, 1 день, за час)
- Загрузка файлов (фото документов, сканы) с телефона
- Открытие внешних ссылок в браузере

### 2.2. Экран входа

- Поля логин и пароль
- Кнопка «Запомнить меня»
- Вход по биометрии (если включено)

### 2.3. Главный экран (дашборд)

- Список ближайших заседаний (следующие 7 дней)
- У каждого заседания: номер дела, суд, дата, время, судья
- Красная метка если заседание сегодня
- Pull-to-refresh

### 2.4. Список дел

- Фильтры: по статусу, по суду, по ответственному
- Поиск по номеру дела
- Карточка: истец, ответчик, статус, дата следующего заседания
- Свайп влево/вправо для быстрых действий

### 2.5. Карточка дела

- Вся информация о деле
- Статус с возможностью быстрого изменения
- Вкладки: Основное / Вложения / Исполнительное производство
- Кнопка «Позвонить клиенту»
- Кнопка «Написать email»
- Кнопка «Открыть в суде» (внешняя ссылка)

### 2.6. Добавление/редактирование дела

- Упрощённая форма (ключевые поля)
- Фотографирование документа и прикрепление к делу
- Выбор клиента из справочника

### 2.7. Календарь

- Заседания на сетке месяц/неделя
- При клике на дату — список заседаний
- Экспорт в системный календарь (iOS Calendar / Google Calendar)

### 2.8. Профиль пользователя

- ФИО, роль, email
- Кнопка «Выйти»
- Настройки уведомлений
- Выбор часового пояса

## 3. API (уже реализовано)

| Метод | URL | Описание |
|---|---|---|
| POST | /api/v1/auth/login | Логин, JWT + refresh_token |
| POST | /api/v1/auth/refresh | Обновление JWT |
| GET/PATCH | /api/v1/auth/me | Профиль пользователя |
| POST | /api/v1/auth/change-password | Смена пароля |
| GET | /api/v1/cases | Список дел |
| GET | /api/v1/cases/{id} | Детали дела |
| PATCH | /api/v1/cases/{id}/status | Обновить статус |
| POST | /api/v1/cases/{id}/attachments | Загрузить файл |
| GET | /api/v1/cases/{id}/attachments | Список вложений |
| GET | /api/v1/clients | Список клиентов |
| POST | /api/v1/auth/device-token | Регистрация FCM токена |
| GET | /api/v1/sync | Синхронизация всех данных |
| **GET** | **/api/v1/calendar** | **⬜ нужно добавить** |

## 4. Пуш-уведомления (FCM)

| Событие | Текст уведомления |
|---|---|
| Заседание через 3 дня | Напоминание: заседание по делу N через 3 дня |
| Заседание завтра | Завтра заседание по делу N в 14:30, зал 305 |
| Заседание через час | Через час заседание по делу N |
| Статус дела изменён | Статус дела N изменён на «Выдан ИЛ» |
| Новое вложение | К делу N добавлен новый документ |

- Инфраструктура FCM готова (`push_notification.py`)
- Нужно: Firebase-проект + credentials + APScheduler задачи

---

## 5. Офлайн-режим

- Синхронизируются: дела, метаданные вложений, справочник клиентов
- iOS: реализовано через SwiftData
- Android: Room + `/api/v1/sync` endpoint (реализовано частично)

## 9. Критерии приёмки

1. Приложение устанавливается на iOS 15+ и Android 8+
2. Вход по логину/паролю и по биометрии работает
3. Пуш-уведомления о заседаниях приходят
4. Список дел, карточка, смена статуса работают
5. Можно сфотографировать документ и прикрепить к делу
6. В офлайн-режиме видны ранее загруженные дела
7. При появлении интернета изменения синхронизируются
